Review: What is Access Control?

Access Control / IDOR Vulnerability
Access Control means making sure users can only see and do things they're AUTHORIZED to do.

IDOR (Insecure Direct Object Reference) is when you can access someone else's data by simply changing a value in the URL or form.

Real-World Example:
Your bank account page: bank.com/account?user=1001
What if you change it to: bank.com/account?user=1002?
If you can suddenly see someone ELSE's bank account... that's an IDOR vulnerability!

Review: What are Injection Attacks?

Injection Attack
Injection attacks happen when an attacker types CODE into an input field (like a login form), and the website accidentally RUNS that code.

SQL Injection Example:
Normal login: Username = admin, Password = mypassword
The website checks: SELECT * FROM users WHERE user='admin' AND pass='mypassword'

Attack: Username = admin' --
The query becomes: SELECT * FROM users WHERE user='admin' --' AND pass='...'
The -- is a SQL comment - it ignores the password check entirely!

Quick Security Review Quiz

A website shows your profile at /profile?id=42. You change it to /profile?id=43 and see someone else's private data. What vulnerability is this?

A) SQL Injection - you're injecting database commands
B) IDOR / Broken Access Control - you're accessing unauthorized data
C) Phishing - you're tricking users with fake pages
D) XSS - you're injecting JavaScript
Remember: IDOR = changing IDs in URLs to see other people's stuff. Injection = putting code in input fields that gets executed.
