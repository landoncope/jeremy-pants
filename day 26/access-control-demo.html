What is Access Control / IDOR?

IDOR stands for Insecure Direct Object Reference. It happens when a website uses user-controllable input (like a URL parameter) to access data without properly checking if the user is authorized.

How Would a Real Website Prevent This?

  • Session-based authentication: The server should check who is logged in (via session/cookies) and ONLY show that user's data - not just trust whatever user ID is in the URL
  • Authorization checks: Before returning any data, verify: "Is this user ALLOWED to see this data?"
  • Don't use predictable IDs: Instead of user=1001, 1002, 1003... use random UUIDs like user=a7f3b2c1-9d4e-4f8a-b123-456789abcdef
  • Rate limiting: Detect when someone is trying many different user IDs rapidly
