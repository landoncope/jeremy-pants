Vocab Review üìö

Streaming

Quick Reminder: A method of transmitting data (especially video and audio) over a network as a continuous flow, allowing playback to begin while the rest is still being received.

Think of it like: Watching a YouTube video that starts playing right away - you don't wait for the entire video to download!

Examples:

  • Netflix, YouTube, Disney+, Spotify
  • Twitch live streams
  • Video calls on Zoom or FaceTime

New Vocabulary üìö

Phishing

Definition: A type of online scam where criminals send fake emails, texts, or messages pretending to be from legitimate companies or people to trick you into giving away passwords, credit card numbers, or other personal information.

Real-world: Imagine someone wearing a fake UPS uniform knocking on your door asking for your house keys "for a delivery." That's what phishing does online - criminals dress up as trusted companies to steal your info!

Common Phishing Tricks:

  • üé£ Fake emails claiming your account will be "suspended"
  • üé£ Texts saying you "won a prize" - just click this link!
  • üé£ Messages with urgent language: "ACT NOW!" or "VERIFY IMMEDIATELY!"
  • üé£ Links that look almost right (paypa1.com instead of paypal.com)
üö® Red Flags: Spelling errors, generic greetings ("Dear Customer"), threats of account closure, requests for passwords or credit cards. Legitimate companies NEVER ask for passwords via email!

New Vocabulary üìö

Malware

Definition: Malicious software designed to damage, disrupt, or steal data from your computer, phone, or network. It's the umbrella term for all bad software including viruses, ransomware, spyware, and trojans.

Real-world: Think of it as digital germs or pests! Just like how bacteria makes you sick or termites destroy a house, malware infects your device to steal info, spy on you, or hold your files hostage.

Types of Malware:

  • ü¶† Virus: Copies itself to spread to other files/computers
  • üîí Ransomware: Locks your files and demands payment to unlock them
  • üëÅÔ∏è Spyware: Secretly watches what you do and steals passwords
  • üê¥ Trojan: Pretends to be legit software but does bad stuff in the background
  • ü§ñ Bot: Takes control of your device to attack others
‚ö†Ô∏è How to avoid malware: Don't download sketchy apps, keep software updated, use antivirus software, and don't click suspicious links or email attachments!

Step 1: Make Sure Your File is Named Correctly

‚ö†Ô∏è CRITICAL: Your main file MUST be named index.html and be in the root directory (not in a subfolder)!

Check Your File:

  1. Look at the Explorer panel in Codespaces (left sidebar)
  2. Find your main HTML file
  3. ‚úÖ CORRECT: index.html (at the root level)
  4. ‚ùå WRONG: game.html or src/index.html
üí° If your file is named wrong: Right-click it ‚Üí Rename ‚Üí Change to "index.html"
üí° If it's in a subfolder: Drag it to the root level (top of Explorer panel)

Deploy Your Game! üöÄ

Step 2: Commit and Push Your Code

We need to save your code to GitHub before we can deploy it.

In the Codespaces Terminal:

git add . git commit -m "Add my game" git push

What Each Command Does:

  • git add . - Stages all your files for commit
  • git commit -m "message" - Saves a snapshot with a message
  • git push - Uploads your code to GitHub
üí° Pro Tip: You can change "Add my game" to any message that describes your changes, like "Fix snake speed" or "Add sound effects"

Deploy Your Game! üöÄ

Step 3: Enable GitHub Pages

  1. Go to your repository on GitHub.com
    • Open a new tab and go to github.com
    • Click on your repository name
  2. Click "Settings" (in the top menu bar)
  3. Click "Pages" (in the left sidebar)
  4. Under "Source":
    • Select "Deploy from a branch"
    • Select "main" branch
    • Leave folder as "/ (root)"
  5. Click "Save"
‚è±Ô∏è Be Patient: It takes 1-2 minutes for your site to deploy. Don't panic if it doesn't work immediately!

Troubleshooting üîß

Common Issues and Fixes:

‚ùå Problem: Blank page or 404 error

Solutions:

  • Make sure file is named index.html (lowercase, no typos)
  • Make sure index.html is in root directory, NOT in a subfolder
  • Did you push your code? Run: git push
  • Wait 2-3 minutes, then try again

‚ùå Problem: Old version showing (changes not appearing)

Solutions:

  • Clear your browser cache (Ctrl+Shift+R or Cmd+Shift+R)
  • Open in incognito/private window
  • Make sure you ran git add . then git commit then git push
